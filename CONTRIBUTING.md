# Contributing to THE-DRAGGY-ENCODER

First off, thank you for considering contributing to **The-Draggy-Encoder**! It's people like you that make it a great tool for everyone.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct:
- Be respectful and inclusive.
- Focus on what is best for the community.
- Gracefully accept constructive criticism.

## How to Contribute

### 1. Reporting Bugs
- Use the **GitHub Issue Tracker**.
- Check if the issue has already been reported.
- Include steps to reproduce, your OS, and your hardware (GPU model).

### 2. Suggesting Enhancements
- Open an issue with the [Feature Request] tag.
- Explain why the feature would be useful and how it should work.

### 3. Pull Requests
1.  **Fork** the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  If you've changed APIs, update the documentation.
4.  Ensure the test suite passes.
5.  Make sure your code lints (we use `flake8` or `black` standards).
6.  Issue that pull request!

## Development Setup

1.  Follow the **Build from Source** section in [README.md](README.md).
2.  Use a virtual environment (`venv`).
3.  Install development dependencies: `pip install -r requirements.txt`.

## Style Guidelines

- **Python**: Follow PEP 8.
- **UI**: Use Qt (PySide6) layouts instead of absolute positioning.
- **Variables**: Use snake_case for functions/variables and PascalCase for classes.
